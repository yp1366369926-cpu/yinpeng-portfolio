# HMI 概念稿分享：设计说明

## 设计思路

本次详情页以“从功能界面，到可被驾驶的数字空间”为主线，首屏保留原有沉浸式座舱视觉，首屏之后改用六个章节重新组织项目内容。叙事顺序从概念原则进入体验系统，再落到视觉语言、安全信息流、空间化体验和系统输出。

核心设计骨架是“状态—任务—情绪”：状态帮助驾驶者快速确认车辆，任务让高频操作保持清晰，情绪通过 3D 场景、光效和动效建立品牌记忆。游戏化视觉只作为体验增强手段，安全性和驾驶专注始终是底层原则。

## 页面结构

- 设计概念：说明游戏化 3D HMI 的设计边界。
- 核心体验系统：展示首页、驾驶模式、能源、媒体与空调等关键任务。
- 视觉语言：说明车辆模型、光效与品牌色彩的使用方式。
- 安全与信息流：强调辅助驾驶、提示和反馈的出现时机。
- 空间化体验：把座舱看作人与车共同使用的数字空间。
- 系统输出：总结一套可以继续扩展到不同场景的 HMI 设计系统。

## 展示原则

使用大图、章节编号和短句说明建立浏览节奏；界面图片不再按素材顺序堆叠，而是围绕设计判断进行编排。桌面端使用错位大图增强空间感，移动端自动切换为单列阅读，保证信息层级和图片细节可读。
